# Security Policy

## Supported Versions

Only the latest major version receives security updates.

| Version | Supported |
|---------|-----------|
| 1.x     | ✅ Yes    |
| 0.x     | ⚠️ LTS    |

LTS (Long-term Support) means we'll patch critical security issues only.

## Reporting a Vulnerability

**Do not** open a public GitHub issue for security vulnerabilities.

Instead, please email: security@lomapper.dev

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Proposed fix (if any)

We will acknowledge receipt within 48 hours and aim to release a patch within 1 week for critical issues.

## Security Practices

LoMapper follows these security principles:
- ✅ All code is subject to security review before release
- ✅ Dependencies are regularly audited for vulnerabilities
- ✅ Generated code follows OWASP guidelines
- ✅ Sensitive operations are flagged with security warnings

## Disclaimer

LoMapper is provided "as is" without warranty. While we strive for secure code, users are responsible for evaluating its suitability for their security requirements.
